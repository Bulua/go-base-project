# GoBaseProject 自研后台基座完整开发方案

## 1. 项目定位

GoBaseProject，中文名“GoBaseProject”，是一个面向安全类业务系统的自研后台基座。它不依赖 gin-vue-admin，只参考常见后台系统能力，沉淀认证、权限、菜单、配置和审计能力。

## 2. 第一阶段功能范围

| 模块 | 功能 |
|---|---|
| 认证鉴权 | 登录、退出、JWT 签发、JWT 黑名单、当前用户信息 |
| 用户管理 | 用户增删改查、冻结、重置密码、分配角色 |
| 角色管理 | 角色增删改查、角色树、菜单授权、API 授权、数据权限 |
| 菜单路由 | 菜单树、动态路由、路由参数、隐藏路由、外链 |
| 按钮权限 | 菜单按钮定义、角色按钮授权、前端按钮控制 |
| API 权限 | API 资源维护、白名单、权限策略、接口鉴权 |
| 数据权限 | 基于角色的数据可见范围控制 |
| 字典管理 | 字典主表、字典项、树形字典 |
| 系统参数 | 系统级配置、开关、运行参数 |
| 审计日志 | 登录日志、操作日志、失败原因、请求耗时 |

## 3. 核心调用链

```text
登录 -> 获取 Token -> 获取用户信息 -> 获取角色 -> 获取菜单树 -> 前端生成动态路由 -> 请求 API -> 后端鉴权 -> 数据权限过滤 -> 记录操作日志
```

## 4. 推荐技术栈

后端建议：Go + Gin + GORM + MySQL + Redis + JWT + Viper + Zap。

前端建议：Vue3 + TypeScript + Vite + Element Plus + Pinia + Vue Router + Axios。

## 6. 开发顺序

1. 数据库结构和初始化数据
2. 后端基础框架
3. 统一响应和错误码
4. JWT 认证中间件
5. 登录接口
6. 用户管理
7. 角色管理
8. 菜单管理和动态路由
9. API 资源和权限策略
10. 按钮权限
11. 数据权限
12. 字典和参数
13. 审计日志
14. 前端权限联动
