# 01-认证鉴权模块

## 1. 模块职责

负责系统入口安全，包括登录、退出、Token 签发、Token 黑名单、当前用户上下文、动态菜单和按钮权限获取。

## 2. 涉及数据表

```text
gbp_users
gbp_roles
gbp_user_roles
gbp_jwt_blocklist
gbp_login_audit_logs
gbp_role_menus
gbp_menus
gbp_role_actions
gbp_menu_actions
```

## 3. 核心接口

| 方法 | 路径 | 说明 | 鉴权 |
|---|---|---|---|
| POST | /api/v1/auth/login | 用户登录 | 否 |
| POST | /api/v1/auth/logout | 退出登录 | 是 |
| POST | /api/v1/auth/refresh | 刷新 Token | 是/可选 |
| GET | /api/v1/auth/profile | 当前用户信息 | 是 |
| GET | /api/v1/auth/routes | 当前用户动态路由 | 是 |
| GET | /api/v1/auth/actions | 当前用户按钮权限 | 是 |

## 4. 关键流程

1. 接收账号密码
2. 查询用户并校验状态
3. 校验密码哈希
4. 查询用户角色
5. 签发 JWT
6. 写入登录日志
7. 返回 Token 和用户信息

## 5. 开发注意事项

1. JWT 黑名单只存 token_hash，不存原始 Token。
2. 登录失败也必须记录登录日志。
3. 密码建议使用 bcrypt 或 argon2id。
4. 退出登录时将当前 Token 摘要写入黑名单。
