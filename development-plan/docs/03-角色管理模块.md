# 03-角色管理模块

## 1. 模块职责

负责 RBAC 权限核心模型，包括角色管理、角色状态、角色菜单授权、角色按钮授权、角色 API 授权和角色数据权限配置。

## 2. 涉及数据表

```text
gbp_roles
gbp_user_roles
gbp_role_menus
gbp_role_actions
gbp_permission_policies
gbp_role_data_scopes
gbp_operation_audit_logs
```

## 3. 核心接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/v1/roles | 角色列表 |
| GET | /api/v1/roles/tree | 角色树 |
| GET | /api/v1/roles/:id | 角色详情 |
| POST | /api/v1/roles | 创建角色 |
| PUT | /api/v1/roles/:id | 修改角色 |
| DELETE | /api/v1/roles/:id | 删除角色 |
| PUT | /api/v1/roles/:id/menus | 分配菜单 |
| PUT | /api/v1/roles/:id/actions | 分配按钮 |
| PUT | /api/v1/roles/:id/apis | 分配 API |
| PUT | /api/v1/roles/:id/data-scopes | 分配数据权限 |

## 4. 关键流程

```text
角色授权请求入参校验 -> 删除旧授权 -> 写入新授权 -> 刷新权限缓存 -> 记录操作日志
```

## 5. 开发注意事项

1. super_admin 不允许删除。
2. 已绑定用户的角色不建议直接删除。
3. 删除父角色时需要处理子角色。
4. 菜单、按钮、API、数据权限授权应使用事务。
