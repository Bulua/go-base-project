# 04-菜单路由模块

## 1. 模块职责

负责菜单维护、前端动态路由生成、菜单路由参数配置、隐藏路由和外链路由配置。

## 2. 涉及数据表

```text
gbp_menus
gbp_menu_route_params
gbp_role_menus
gbp_menu_actions
gbp_role_actions
```

## 3. 核心接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/v1/menus | 菜单列表 |
| GET | /api/v1/menus/tree | 菜单树 |
| GET | /api/v1/menus/:id | 菜单详情 |
| POST | /api/v1/menus | 创建菜单 |
| PUT | /api/v1/menus/:id | 修改菜单 |
| DELETE | /api/v1/menus/:id | 删除菜单 |
| GET | /api/v1/menus/:id/params | 路由参数列表 |
| POST | /api/v1/menus/:id/params | 创建路由参数 |
| DELETE | /api/v1/menus/params/:paramId | 删除路由参数 |

## 4. 关键流程

```text
用户角色 -> 查询角色菜单 -> 查询菜单详情 -> 过滤禁用/删除 -> 按 parent_id 组装菜单树 -> 前端生成动态路由
```

## 5. 开发注意事项

1. 新增菜单时校验父菜单存在。
2. 删除菜单时同步清理路由参数和按钮定义。
3. component_path 与前端组件路径映射。
4. 隐藏路由不展示在菜单中，但仍注册路由。
