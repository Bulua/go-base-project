# 05-按钮权限模块

## 1. 模块职责

负责页面级操作按钮的权限控制，例如新增、编辑、删除、导出、导入、分配权限等。

## 2. 涉及数据表

```text
gbp_menu_actions
gbp_role_actions
gbp_menus
gbp_roles
```

## 3. 核心接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/v1/menus/:menuId/actions | 菜单按钮列表 |
| POST | /api/v1/menus/:menuId/actions | 创建按钮 |
| PUT | /api/v1/menu-actions/:id | 修改按钮 |
| DELETE | /api/v1/menu-actions/:id | 删除按钮 |
| GET | /api/v1/roles/:id/actions | 查询角色按钮权限 |
| PUT | /api/v1/roles/:id/actions | 分配角色按钮权限 |

## 4. 关键流程

```text
菜单按钮定义 -> 角色按钮授权 -> 登录后获取按钮权限 -> 前端 v-auth 控制按钮显示
```

## 5. 开发注意事项

1. 按钮权限只控制前端显示，不能替代 API 鉴权。
2. action_code 建议使用 add/edit/delete/export/import 等短编码。
3. 前端可组合为 system:user:add 这类权限字符串。
