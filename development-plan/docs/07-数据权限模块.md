# 07-数据权限模块

## 1. 模块职责

第一阶段采用基于角色的数据范围模型，控制当前角色可以查看哪些角色创建或拥有的数据。

## 2. 涉及数据表

```text
gbp_roles
gbp_role_data_scopes
```

## 3. 核心接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/v1/roles/:id/data-scopes | 查询角色数据权限 |
| PUT | /api/v1/roles/:id/data-scopes | 分配角色数据权限 |

## 4. 关键流程

```text
当前用户角色 -> 查询 visible_role_ids -> 业务表 created_role_id IN (...) -> 返回授权范围数据
```

## 5. 开发注意事项

1. 业务表建议保留 created_by、created_role_id。
2. 第一阶段不引入组织架构。
3. 第二阶段可扩展租户、部门、数据规则表达式。
