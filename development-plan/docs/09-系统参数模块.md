# 09-系统参数模块

## 1. 模块职责

负责维护系统运行配置，例如系统名称、Token 过期时间、审计开关、登录安全策略等。

## 2. 涉及数据表

```text
gbp_system_params
```

## 3. 核心接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/v1/system-params | 参数列表 |
| GET | /api/v1/system-params/:key | 根据 key 查询参数 |
| POST | /api/v1/system-params | 创建参数 |
| PUT | /api/v1/system-params/:id | 修改参数 |
| DELETE | /api/v1/system-params/:id | 删除参数 |

## 4. 关键流程

```text
管理员维护参数 -> 服务启动/运行时读取参数 -> 按 key 缓存 -> 配置变更后刷新缓存
```

## 5. 开发注意事项

1. 敏感参数建议 is_encrypted=1。
2. 前端查询敏感参数时不能返回明文。
3. 修改关键安全参数需要写入审计日志。
