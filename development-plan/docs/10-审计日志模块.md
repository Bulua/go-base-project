# 10-审计日志模块

## 1. 模块职责

记录登录行为和后台操作行为，用于安全追踪、问题排查和合规审计。

## 2. 涉及数据表

```text
gbp_login_audit_logs
gbp_operation_audit_logs
```

## 3. 核心接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/v1/audit/login-logs | 登录日志分页 |
| GET | /api/v1/audit/operation-logs | 操作日志分页 |
| GET | /api/v1/audit/operation-logs/:id | 操作日志详情 |
| DELETE | /api/v1/audit/login-logs/cleanup | 清理登录日志 |
| DELETE | /api/v1/audit/operation-logs/cleanup | 清理操作日志 |

## 4. 关键流程

```text
请求进入 -> 记录开始时间 -> 执行业务 -> 获取状态码和耗时 -> 脱敏请求/响应 -> 异步写入操作日志
```

## 5. 开发注意事项

1. 密码、Token、密钥等字段必须脱敏。
2. 文件上传接口不记录完整请求体。
3. 日志数据可按月份分表。
