# 12-前端开发规范

## 1. 技术选型

前端建议使用 Vue3 + TypeScript + Vite + Element Plus + Pinia + Vue Router + Axios。第一阶段重点完成登录、动态路由、菜单渲染、按钮权限、基础 CRUD 页面和审计查询。

## 3. 放置约定

```text
api：接口请求封装，request 放 Axios 实例、拦截器和错误处理
views：页面级组件，按菜单结构和业务模块拆分
store：Pinia 状态，按模块拆分
types：接口 DTO、页面模型、枚举类型
components/common：纯通用组件，例如表格工具栏、搜索表单、分页容器
components/business：带业务语义的复合组件，例如角色选择器、菜单树选择器
components/layout：布局相关组件，例如侧边栏、顶部栏、面包屑
composables：可复用组合函数，例如 useDict、usePermission、useTable
directives：自定义指令，例如 v-auth
permissions：权限常量、按钮权限判断和动态路由转换
utils：纯工具函数，不直接依赖业务状态
constants：跨模块常量
assets/styles：全局样式、变量、主题覆盖
```

## 4. 命名约定

1. 页面组件使用 `index.vue` 作为模块入口，例如 `views/system/user/index.vue`。
2. 业务组件使用 PascalCase，例如 `RoleSelect.vue`、`MenuTreeSelect.vue`。
3. 组合函数使用 `useXxx.ts`，例如 `useDict.ts`、`usePermission.ts`。
4. API 文件按资源命名，例如 `api/user/index.ts`、`api/role/index.ts`。
5. 类型文件按模块命名，例如 `types/user/index.ts`。

## 5. 权限约定

前端权限只负责交互层展示控制，不能替代后端鉴权。

```text
菜单权限：由 /api/v1/auth/routes 返回动态路由树
按钮权限：由 /api/v1/auth/actions 返回权限点列表
API 权限：后端中间件最终判定
```

按钮权限建议统一为：

```text
system:user:add
system:user:edit
system:role:assign_menu
```

页面中通过 `v-auth` 指令或 `usePermission` 判断是否展示操作入口。

## 6. 请求约定

1. 所有接口通过 `api/request` 中的 Axios 实例发起。
2. 请求拦截器统一注入 Token、Trace ID 和租户/环境信息。
3. 响应拦截器统一处理业务错误码、401 退出、403 无权限和消息提示。
4. 列表接口统一使用 `page`、`page_size`，响应统一适配为 `items`、`total`。
